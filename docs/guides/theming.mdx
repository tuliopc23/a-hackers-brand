# Theming & Customization

A Hacker's Brand provides a powerful theming system built on CSS custom properties, allowing you to customize colors, effects, and styling across all three design variants.

## CSS Custom Properties

The theming system is built on CSS custom properties (CSS variables) that can be customized at the root level or per component.

### Terminal Theme Variables

```css
:root {
  /* Core terminal colors */
  --terminal-green: #00ff41;
  --terminal-cyan: #00ffff;
  --terminal-blue: #0080ff;
  --terminal-yellow: #ffff00;
  --terminal-red: #ff0040;
  
  /* Background colors */
  --terminal-bg: #000000;
  --terminal-bg-secondary: #0a0a0a;
  --terminal-border: #00ff41;
  
  /* Effects */
  --terminal-glow: 0 0 5px var(--terminal-green);
  --terminal-glow-strong: 0 0 10px var(--terminal-green), 0 0 20px var(--terminal-green);
  
  /* Typography */
  --terminal-font-family: 'JetBrains Mono', 'Fira Code', 'Courier New', monospace;
  --terminal-font-size: 14px;
  --terminal-line-height: 1.4;
}
```

## Customizing Themes

### Basic Customization

Override CSS custom properties to customize the appearance:

```css
/* Custom terminal theme */
:root {
  --terminal-green: #39ff14; /* Brighter green */
  --terminal-glow: 0 0 15px var(--terminal-green); /* Stronger glow */
  --terminal-bg: #0d1117; /* GitHub dark background */
}
```

### Component-Specific Theming

Target specific components with custom styling:

```css
/* Custom button styling */
.custom-button {
  --terminal-green: #ff6b35;
  --glass-radius: 24px;
  --liquid-glow-intensity: 1.2;
}
```

```svelte
<Button variant="terminal" class="custom-button">
  Custom Styled Button
</Button>
```

### Dynamic Theme Switching

Create a theme switcher component:

```svelte
<script>
  import { Button } from 'a-hacker-brand';
  
  let currentTheme = 'terminal';
  
  const themes = {
    terminal: {
      '--terminal-green': '#00ff41',
      '--terminal-bg': '#000000'
    },
    matrix: {
      '--terminal-green': '#39ff14',
      '--terminal-bg': '#0d1117'
    },
    amber: {
      '--terminal-green': '#ffb000',
      '--terminal-bg': '#1a1000'
    }
  };
  
  function applyTheme(themeName) {
    const theme = themes[themeName];
    const root = document.documentElement;
    
    Object.entries(theme).forEach(([property, value]) => {
      root.style.setProperty(property, value);
    });
    
    currentTheme = themeName;
  }
</script>

<div class="theme-switcher">
  {#each Object.keys(themes) as theme}
    <Button 
      variant="terminal"
      class:active={currentTheme === theme}
      on:click={() => applyTheme(theme)}
    >
      {theme}
    </Button>
  {/each}
</div>
```

## Advanced Theming

### Creating Custom Variants

You can create your own variant by extending the existing system:

```css
/* Custom "retro" variant */
:root {
  --retro-amber: #ffb000;
  --retro-green: #00ff00;
  --retro-bg: #2d1b00;
  --retro-scanlines: repeating-linear-gradient(
    0deg,
    transparent,
    transparent 2px,
    rgba(255, 176, 0, 0.03) 2px,
    rgba(255, 176, 0, 0.03) 4px
  );
}

.retro-variant {
  background: var(--retro-bg);
  color: var(--retro-amber);
  position: relative;
}

.retro-variant::before {
  content: '';
  position: absolute;
  top: 0;
  left: 0;
  right: 0;
  bottom: 0;
  background: var(--retro-scanlines);
  pointer-events: none;
}
```

### Responsive Theming

Adapt themes based on screen size:

```css
/* Mobile-first theming */
:root {
  --glass-radius: 8px;
  --terminal-font-size: 12px;
}

@media (min-width: 768px) {
  :root {
    --glass-radius: 16px;
    --terminal-font-size: 14px;
  }
}

@media (min-width: 1024px) {
  :root {
    --glass-radius: 24px;
    --terminal-font-size: 16px;
  }
}
```

### Dark/Light Mode Support

While A Hacker's Brand is primarily designed for dark interfaces, you can create light variants:

```css
/* Light mode overrides */
@media (prefers-color-scheme: light) {
  :root {
    --terminal-bg: #f8f9fa;
    --terminal-green: #28a745;
    --glass-bg: rgba(0, 0, 0, 0.1);
    --glass-border: rgba(0, 0, 0, 0.2);
  }
}

/* Manual light mode class */
.light-mode {
  --terminal-bg: #ffffff;
  --terminal-green: #007acc;
  --glass-bg: rgba(0, 0, 0, 0.05);
}
```

## Animation & Motion Theming

### Motion Preferences

Respect user motion preferences:

```css
/* Reduced motion support */
@media (prefers-reduced-motion: reduce) {
  :root {
    --liquid-animation-duration: 0s;
    --terminal-glow: none;
    --glass-blur: none;
  }
  
  * {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
}
```

### Custom Animations

Define custom animation themes:

```css
/* High-performance mode */
.performance-mode {
  --liquid-glow-intensity: 0.3;
  --glass-blur: blur(5px);
  --terminal-glow: 0 0 2px var(--terminal-green);
}

/* High-impact mode */
.high-impact-mode {
  --liquid-glow-intensity: 1.5;
  --glass-blur: blur(20px);
  --terminal-glow: 0 0 20px var(--terminal-green), 0 0 40px var(--terminal-green);
}
```

## Best Practices

<AccordionGroup>
  <Accordion title="Performance Considerations">
    - Use CSS custom properties for dynamic theming instead of JavaScript
    - Limit the number of simultaneous glow effects
    - Consider `prefers-reduced-motion` for accessibility
    - Test performance on lower-end devices
  </Accordion>
  
  <Accordion title="Accessibility Guidelines">
    - Maintain sufficient contrast ratios (WCAG 2.1 AA: 4.5:1)
    - Provide high-contrast alternatives
    - Test with screen readers
    - Ensure focus indicators are visible
  </Accordion>
  
  <Accordion title="Consistency">
    - Use a consistent color palette across variants
    - Maintain proportional spacing and sizing
    - Keep animation durations consistent
    - Document custom theme variables
  </Accordion>
</AccordionGroup>

## Theme Utilities

### CSS-in-JS Integration

For frameworks that use CSS-in-JS:

```javascript
// Styled-components example
import styled from 'styled-components';

const ThemedButton = styled.button`
  background: var(--terminal-bg);
  color: var(--terminal-green);
  border: 1px solid var(--terminal-green);
  box-shadow: var(--terminal-glow);
  
  &:hover {
    box-shadow: var(--terminal-glow-strong);
  }
`;
```

### Tailwind CSS Integration

Extend Tailwind with A Hacker's Brand colors:

```javascript
// tailwind.config.js
module.exports = {
  theme: {
    extend: {
      colors: {
        'terminal': {
          green: 'var(--terminal-green)',
          cyan: 'var(--terminal-cyan)',
          blue: 'var(--terminal-blue)',
        },
        'glass': {
          pink: 'var(--glass-pink)',
          purple: 'var(--glass-purple)',
          blue: 'var(--glass-blue)',
        }
      },
      borderRadius: {
        'glass': 'var(--glass-radius)',
      },
      boxShadow: {
        'terminal': 'var(--terminal-glow)',
        'glass': 'var(--glass-shadow)',
      }
    }
  }
}
```
