# StatusBar

The `StatusBar` component provides a terminal-inspired status bar for displaying file information, system status, and other indicators. It's commonly used at the bottom of the application to show contextual information.

## Usage

```svelte
<script>
  import { StatusBar } from 'a-hacker-brand';
</script>

<StatusBar 
  status="Ready"
  fileType="TypeScript"
  lineNumber={42}
  columnNumber={10}
  encoding="UTF-8"
  variant="terminal"
/>
```

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `status` | `string` | `'Ready'` | Primary status text |
| `encoding` | `string` | `'UTF-8'` | Text encoding information |
| `position` | `string` | `'1:1'` | Cursor position (mobile view) |
| `lineNumber` | `number` | `1` | Current line number |
| `columnNumber` | `number` | `1` | Current column number |
| `fileType` | `string` | `'TypeScript'` | Type of file being edited |
| `variant` | `'terminal' \| 'bubbleTea' \| 'glass'` | `'bubbleTea'` | Visual style variant |
| `size` | `'sm' \| 'md'` | `'md'` | Size of the status bar |
| `indicators` | `Array<{ label: string; value: string; type?: 'normal' \| 'warning' \| 'error' \| 'success' }>` | `[]` | Status indicators to display |
| `autoLayout` | `boolean` | `true` | Whether to automatically adjust body padding |
| `class` | `string` | `''` | Additional CSS classes |
| `children` | `Snippet` | `undefined` | Custom content for the right side |

## Indicator Types

The `indicators` prop accepts an array of objects with the following properties:

- `label`: The name of the indicator
- `value`: The value to display
- `type`: Visual style of the indicator
  - `normal`: Default style (matches variant)
  - `success`: Green success indicator
  - `warning`: Yellow warning indicator
  - `error`: Red error indicator

## Auto Layout

The `autoLayout` prop allows the component to automatically manage the document layout to prevent content from being hidden behind the fixed status bar:

- When `true`, it adds a CSS custom property `--status-bar-height` to the `:root` element
- This allows your layout to reference this value for padding/margin adjustments
- It uses a ResizeObserver to update this value when the status bar height changes

To utilize this in your layout:

```css
.content-container {
  padding-bottom: var(--status-bar-height, 2.5rem);
}
```

## Responsive Behavior

The StatusBar is responsive and adapts to different screen sizes:

- On small screens, it simplifies to show just essential information
- The indicators section is hidden on small screens and shown on medium and larger screens
- Line/column information is condensed to a simple position string on mobile

## CSS Customization

The StatusBar can be styled with custom classes:

```svelte
<StatusBar
  class="shadow-lg border-t-2"
  variant="terminal"
  status="Custom Styled"
/>
```

    indicators={[
      { label: 'Status', value: 'Connected', type: 'success' }

## Animation Effects

The StatusBar includes subtle animation effects based on the variant:

- **Bubble Tea**: Gentle pulsing effect that subtly changes brightness
- **Terminal**: Static with hover effects on interactive elements
- **Glass**: Minimal hover effects with blur adjustments

## Accessibility

The StatusBar is designed with accessibility in mind:

- Uses `role="status"` to indicate its purpose to screen readers
- Sets `aria-live="polite"` to announce changes to the status
- Ensures color contrast meets WCAG AA standards
- Responsively adapts to different screen sizes
