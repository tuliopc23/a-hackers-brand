# Button

The cornerstone of any cyberpunk interface. Our Button component delivers that authentic terminal feel with customizable neon glows, hover effects, and multiple variants.

## Import

```svelte
import { Button } from '@org/ui'
```

## Advanced Examples

### Loading State

```svelte
<script>
  import { Button, Spinner } from '@org/ui'
  
  let loading = false
  
  async function handleClick() {
    loading = true
    await new Promise(resolve => setTimeout(resolve, 2000))
    loading = false
  }
</script>

<Button 
  variant="cyberpunk" 
  {loading}
  onClick={handleClick}
>
  {#if loading}
    <Spinner size="sm" />
    Hacking...
  {:else}
    Initiate Hack
  {/if}
</Button>
```

### With Icons

```svelte
<script>
  import { Button, Icon } from '@org/ui'
</script>

<Button variant="neon">
  <Icon name="terminal" />
  Open Terminal
</Button>

<Button variant="cyberpunk">
  Access Database
  <Icon name="arrow-right" />
</Button>
```

### Button Group

```svelte
<script>
  import { Button, ButtonGroup } from '@org/ui'
</script>

<ButtonGroup>
  <Button variant="ghost">Cancel</Button>
  <Button variant="neon">Execute</Button>
  <Button variant="cyberpunk">Deploy</Button>
</ButtonGroup>
```

## Props

| Property | Type | Default | Description |
|----------|------|---------|-------------|
| `variant` | `'default' \| 'cyberpunk' \| 'neon' \| 'ghost' \| 'glitch'` | `'default'` | Button style variant |
| `size` | `'sm' \| 'md' \| 'lg'` | `'md'` | Button size |
| `color` | `string` | `'green'` | Neon glow color for neon variant |
| `disabled` | `boolean` | `false` | Disable button interaction |
| `loading` | `boolean` | `false` | Show loading state |
| `type` | `'button' \| 'submit' \| 'reset'` | `'button'` | HTML button type |
| `href` | `string` | `undefined` | If provided, renders as link |
| `onClick` | `function` | `undefined` | Click handler function |

## Events

| Event | Description |
|-------|-------------|
| `click` | Fired when button is clicked |
| `focus` | Fired when button receives focus |
| `blur` | Fired when button loses focus |
| `mouseenter` | Fired when mouse enters button |
| `mouseleave` | Fired when mouse leaves button |

  --btn-neon-glow: 0 0 20px currentColor;

```svelte

  class="custom-button"

  Custom Style
</Button>

<style>
  :global(.custom-button) {

  }

```

## Best Practices

<Tip>
Use `variant="cyberpunk"` for primary actions and `variant="ghost"` for secondary actions to maintain visual hierarchy.
</Tip>

<Warning>
Avoid using too many neon buttons in one interface - it can be overwhelming. Mix variants for better UX.
</Warning>

<Info>
The button automatically adjusts its contrast based on the background color for accessibility compliance.
</Info>
