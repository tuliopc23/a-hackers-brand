# Installation Guide

Get A Hacker's Brand up and running in your project. Multiple installation methods available for different setups.

## Package Installation

<CodeGroup>

```bash bun
bun add a-hacker-brand
```

```bash npm
npm install a-hacker-brand
```

```bash yarn
yarn add a-hacker-brand
```

```bash pnpm
pnpm add a-hacker-brand
```

</CodeGroup>

## Peer Dependencies

A Hacker's Brand requires several peer dependencies for full functionality:

<CodeGroup>

```bash bun
bun add svelte @threlte/core @threlte/extras three lucide-svelte
```

```bash npm
npm install svelte @threlte/core @threlte/extras three lucide-svelte
```

```bash yarn
yarn add svelte @threlte/core @threlte/extras three lucide-svelte
```

```bash pnpm
pnpm add svelte @threlte/core @threlte/extras three lucide-svelte
```

</CodeGroup>

<Note>
**WebGL components are optional**: If you don't plan to use 3D components like `GlassScene` or `LazyThrelteCanvas`, you can skip the Threlte and Three.js dependencies.
</Note>

## Framework Setup

### SvelteKit (Recommended)

A Hacker's Brand is built specifically for Svelte 5 and SvelteKit. Follow these steps for optimal integration:

<Steps>
  <Step title="Install Dependencies">
    ```bash
    bun add a-hacker-brand svelte @threlte/core @threlte/extras three lucide-svelte
    ```
  </Step>
  
  <Step title="Configure Vite">
    Update your `vite.config.js`:
    ```javascript
    import { sveltekit } from '@sveltejs/kit/vite';
    import { defineConfig } from 'vite';

    export default defineConfig({
      plugins: [sveltekit()],
      define: {
        global: 'globalThis',
      },
      optimizeDeps: {
        include: ['a-hacker-brand']
      }
    });
    ```
  </Step>
  
  <Step title="Import Components">
    Start using components in your Svelte files:
    ```svelte
    <script>
      import { Button, Card, GlowText } from 'a-hacker-brand';
    </script>

    <Card variant="glass">
      <GlowText color="pink" intensity="intense">
        Welcome to A Hacker's Brand
      </GlowText>
      <Button variant="terminal">Get Started</Button>
    </Card>
    ```
  </Step>
  
  <Step title="Optional: Add Global Styles">
    Add to your `src/app.css` for enhanced theming:
    ```css
    /* Optional: Cyberpunk theme variables */
    :root {
      --terminal-green: #00ff41;
      --glass-pink: #ff6b9d;
      --glass-purple: #c44af2;
      --glass-radius: 16px;
    }
    
    /* Optional: Dark background for terminal aesthetic */
    body {
      background: #0a0a0a;
      color: #00ff41;
    }
    ```
  </Step>
</Steps>

### Vite + Svelte

<Steps>
  <Step title="Install Dependencies">
    ```bash
    bun add a-hacker-brand svelte @threlte/core @threlte/extras three lucide-svelte
    ```
  </Step>
  
  <Step title="Configure Vite">
    Update your `vite.config.js`:
    ```javascript
    import { svelte } from '@sveltejs/vite-plugin-svelte'
    import { defineConfig } from 'vite'

    export default defineConfig({
      plugins: [svelte()],
      define: {
        global: 'globalThis',
      },
      optimizeDeps: {
        include: ['a-hacker-brand']
      }
    })
    ```
  </Step>
  
  <Step title="Import and Use">
    In your Svelte components:
    ```svelte
    <script>
      import { Button, GlowText } from 'a-hacker-brand';
    </script>

    <GlowText variant="terminal" color="matrix">
      System Online
    </GlowText>
    <Button variant="glass">Execute</Button>
    ```
  </Step>
</Steps>

## TypeScript Configuration

For full TypeScript support, ensure your `tsconfig.json` includes:

```json
{
  "compilerOptions": {
    "moduleResolution": "bundler",
    "target": "ES2022",
    "lib": ["ES2022", "DOM", "DOM.Iterable"],
    "types": ["svelte"]
  }
}
```

## Environment Variables

Create a `.env` file for optional configuration:

```bash
# Optional: Enable WebGL components
VITE_ENABLE_WEBGL=true

# Optional: Default variant
VITE_DEFAULT_VARIANT=terminal

# Optional: Performance monitoring
VITE_PERFORMANCE_MONITORING=true
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Module not found errors">
    ```bash
    # Clear cache and reinstall
    rm -rf node_modules package-lock.json bun.lockb
    bun install
    
    # Or with npm
    rm -rf node_modules package-lock.json
    npm install
    ```
  </Accordion>

  <Accordion title="WebGL context errors">
    WebGL components require a modern browser and GPU support:
    - **Chrome 80+** with hardware acceleration enabled
    - **Firefox 78+** with WebGL enabled
    - **Safari 14+** on macOS/iOS
    - **Edge 80+** with hardware acceleration

    If WebGL is not available, components will gracefully fallback to CSS-only versions.
  </Accordion>

  <Accordion title="TypeScript errors">
    ```bash
    # Ensure TypeScript is installed
    bun add -D typescript @tsconfig/svelte
    
    # Check Svelte types
    npx svelte-check --tsconfig ./tsconfig.json
    ```
  </Accordion>

  <Accordion title="Build errors with Vite">
    ```javascript
    // vite.config.js - Add to optimizeDeps
    export default defineConfig({
      optimizeDeps: {
        include: ['a-hacker-brand'],
        exclude: ['@threlte/core', '@threlte/extras'] // If having issues
      },
      define: {
        global: 'globalThis', // Required for Three.js
      }
    })
    ```
  </Accordion>

  <Accordion title="Svelte 5 compatibility">
    A Hacker's Brand requires **Svelte 5.0+**. If you're using an older version:
    
    ```bash
    # Update to Svelte 5
    bun add svelte@^5.0.0
    
    # Update SvelteKit if needed
    bun add @sveltejs/kit@latest
    ```
  </Accordion>
</AccordionGroup>

## Development Setup

For contributing to A Hacker's Brand:

<Steps>
  <Step title="Clone Repository">
    ```bash
    git clone https://github.com/tuliopc23/a-hackers-brand.git
    cd a-hackers-brand
    ```
  </Step>
  
  <Step title="Install Dependencies">
    ```bash
    bun install
    ```
  </Step>
  
  <Step title="Start Development Server">
    ```bash
    bun run dev
    ```
  </Step>
  
  <Step title="Run Tests">
    ```bash
    bun run test
    bun run test:coverage
    ```
  </Step>
  
  <Step title="Build Documentation">
    ```bash
    bun run docs:dev
    ```
  </Step>
</Steps>
